  <name>Task 1: Alerting Service and Configuration</name>
  <files>api/app/services/alerting.py, api/app/config.py, api/.env.example</files>
  <action>
    - Add `alert_webhook_url: str = ""` to `Settings` in `api/app/config.py`.
    - Add `ALERT_WEBHOOK_URL=` to `api/.env.example`.
    - Create `api/app/services/alerting.py` implementing `send_alert(message: str)`.
    - Use synchronous `httpx.post` (as it will be called from worker threads).
    - Implement payload normalization:
      - If "slack.com" in URL -> `{"text": message}`
      - If "discord.com" in URL -> `{"content": message}`
      - Else -> `{"message": message}`
    - Truncate message to 1000 characters to avoid payload limit issues.
    - Wrap the POST in a try/except that logs to `stderr` on failure to prevent alert failures from crashing the caller.
  </action>
  <verify>
    - Check `api/app/config.py` for the new setting.
    - Check `api/app/services/alerting.py` for logic.
  </verify>
  <done>Alerting service is ready for integration and configured via env vars.</done>
