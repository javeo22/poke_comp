# Plan 08-03: Speed Benchmark Integration

## Goal
Integrate dynamic speed benchmarking into the teambuilder so users can immediately see the competitive impact of their speed investment.

## Proposed Changes

### 1. Data Integration
- **Action:** Refactor `StatPointEditor` to fetch dynamic speed tiers from the `/pokemon/speed-tiers` endpoint instead of relying on a hardcoded list.
- **File:** `web/src/components/roster/stat-point-editor.tsx`

### 2. Benchmarking Logic
- **Action:** Implement a comparison function that takes the `finalSpeed` (after points and nature) and finds the closest 5 meta benchmarks (e.g. "Outspeeds Adamant Garchomp", "Outspeed by +1 Dragonite").
- **UI:** Render these benchmarks in a scrolling side-list within the editor.

## Verification Plan
- [ ] **Data Check:** Verify that changing the Nature (e.g. Timid) correctly updates the benchmarks list in real-time.
- [ ] **Accuracy Check:** Compare the results with the `/speed-tiers` page to ensure consistent math.
