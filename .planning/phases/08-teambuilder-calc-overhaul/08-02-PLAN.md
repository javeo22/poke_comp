# Plan 08-02: High-Resolution Meta Tracker

## Goal
Improve the Meta Tracker by surfacing deep usage data (moves, items, roles) directly on the homepage, making the data feel alive and actionable.

## Proposed Changes

### 1. Database (RPC Update)
- **Action:** Update `get_meta_trends` RPC to join with `pokemon_usage`.
- **Logic:** For each trending Pokemon, grab the `moves`, `items`, and `abilities` from the latest `pokemon_usage` entry for that format.
- **Migration:** New SQL file to replace the existing function.

### 2. Backend (Models & Router)
- **Action:** Update `MetaTrendResponse` to include the new usage lists.
- **File:** `api/app/models/meta.py`
- **File:** `api/app/routers/meta.py`

### 3. Frontend (Homepage UI)
- **Action:** Update the "Trending Movers" section to show top 3 moves/items on hover or in a compact card layout.
- **File:** `web/src/app/page.tsx`
