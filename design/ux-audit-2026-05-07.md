# PokeComp UX Audit and Player Workflow Review

Date: 2026-05-07

## Review Scope

Review mode: code analysis plus local runtime HTTP checks. I started the Next.js app on
`http://localhost:3001` and the FastAPI API on `http://127.0.0.1:8000`, then checked the main
routes and API endpoints. I could not run a full visual Playwright audit because Playwright is not
installed in the workspace.

Local runtime checks:

- `GET /`, `/roster`, `/draft`, `/matches` returned `200` from Next.js.
- `GET /health` returned `{"status":"ok"}`.
- Public API data worked for Pokemon, items, usage, trends, and speed tiers.
- Auth-gated endpoints returned `401 Not authenticated` for roster, teams, and matches.
- `GET /meta/latest` returned an empty array, while `GET /usage?format=doubles` returned data.

## What Players Usually Use Today

### Competitive Player Stack

Competitive players typically split work across separate tools:

- Battle simulator and teambuilder: Pokemon Showdown for fast iteration, validation, ladder tests,
  and import/export team text.
- Team sharing: PokePaste, crob.at, team repositories, Discord/Twitter paste links, and tournament
  team pages.
- Meta research: Pokemon HOME Battle Data, Pikalytics, Smogon usage stats, Limitless VGC, VGCPastes
  style repositories, and newer analytics tools that show usage, moves, items, abilities,
  teammates, and trends.
- Damage and speed work: Showdown damage calculator, Pikalytics calc/team builder, custom
  spreadsheets, speed tier charts, and format-specific benchmark lists.
- Matchup prep: opponent's six from team preview, common set lookup, likely lead prediction,
  speed control checks, bring-four notes, and damage rolls against common spreads.
- Match logging: spreadsheets, PASRS-style logs, Showdown replay links, browser extensions,
  notes apps, and team-by-team win-rate tracking.
- Tournament admin: official or event-specific team sheets, RK9 submission flows, pairing pages,
  and opponent view team sheets.

What they value: speed, trust, exact legality, paste interoperability, fast lookup, fewer clicks
during the team-preview timer, and the ability to turn match history into concrete adjustments.

### Casual Player Stack

Casual or new competitive players use a different workflow:

- They start from favorites, rental teams, sample teams, or "what is good right now" lists.
- They need type chart help, simple move/item explanations, role explanations, and clear "why this
  matters" context.
- They are less likely to maintain a complete roster before building a team.
- They value guided presets, beginner-friendly warnings, examples, and low-friction experimentation.
- They need confidence that the app will not make them learn competitive jargon before they see
  value.

What they value: "make a usable team quickly", "explain why", "do not overwhelm me", and "let me
try before account setup".

## Current UX Run-Through

### First Impression

PokeComp has a strong pro-tool identity. The visual direction feels like an esports dashboard and
fits a competitive companion better than a generic Pokedex. The product promise is clear:
roster, team builder, AI draft help, cheatsheets, meta, calc, and match logging.

The problem is that the information architecture exposes too many top-level destinations before
the user has a mental model. The nav has Home, Pokedex, Draft, Calc, Roster, Teams, Cheatsheet,
Meta, Speed, Matches, plus profile/support/auth. For a competitive player this is scannable but
not ordered by match workflow. For a casual player it is dense and jargon-heavy.

Suggested grouping:

- Build: Pokedex, Roster, Teams, Items, Moves.
- Prep: Meta, Speed, Calc, Cheatsheet.
- Battle: Draft.
- Review: Matches, Profile.

### Home

Strengths:

- Strong positioning and immediate "Champions Meta" signal.
- Good use of trends and public stats to make the app feel alive.
- The quick search concept is exactly right for a competitive player.

Issues:

- Critical bug: `HomePage` routes quick draft to `/draft?opp={id}`, but `DraftPage` only reads
  `team`, not `opp`. The quick-start flow lands on Draft with no opponent slot filled.
- The adjacent "Quick Draft" icon button has no `onClick`; selecting from the dropdown triggers
  navigation, but pressing the button does nothing.
- "REGULATION M-A" and "S1 WK 17" are hard-coded UI claims. These need to be data-driven or clearly
  tied to current data freshness.
- First-time users can click into auth-gated tools and hit "Please sign in to continue" rather than
  a value-preview or demo state.
- The home page has a marketing hero, a live board, and a product dashboard feel. That is good for
  conversion, but the fastest competitive path should be one click: "Paste opponent team preview".

Fixes:

- Make `/draft?opp=` populate opponent slot 1, and support multiple names via `?opp=...,...`.
- Add an `onClick` to the Quick Draft button using the current dropdown selection, or remove the
  button if selection is the action.
- Add two first-run CTAs: "Analyze team preview" and "Import Showdown team".
- Add a read-only demo team so users can try Draft, Calc, Cheatsheet, and Match Log before sign-in.

### Onboarding

Strengths:

- Covers the main product flow in the right rough order: Pokedex, Roster, Teams, Draft, Matches.
- The reset button in nav is useful.

Issues:

- The tour says "186 Champions-eligible Pokemon", while the API returned 201. This erodes trust in
  a data-sensitive competitive product.
- The tour is feature-oriented, not task-oriented. Competitive and casual users need different
  first steps.
- It does not send users to a starter action after completion.

Fixes:

- Split onboarding by intent: "I want to build a team", "I want matchup help", "I am learning".
- Replace static counts with live public stats or remove counts from tour copy.
- End on a concrete next step, preferably "Import a Showdown paste" or "Try demo draft".

### Pokedex, Moves, Items, Speed Tiers

Strengths:

- Good reference coverage and clean filters.
- Pokemon detail pages connect to "Add to Roster".
- Speed tiers are valuable and already integrate into stat point editing.
- Moves and items have the expected search/filter table behavior.

Issues:

- `PokemonCard` uses absolutely positioned eligibility markers without a relative parent, so the
  marker can position incorrectly.
- Reference pages are mostly lookup surfaces. They do not answer "should I use this?" quickly
  enough for casual users.
- Moves page truncates effects; that is efficient for dense scanning, but there is no expansion
  affordance.
- Items default `championsOnly` to false while Pokemon and moves default to true. For a Champions
  companion, that inconsistency can confuse users.
- Speed tiers describe "252 EV / 31 IV" even though the rest of the app models Champions stat
  points. That copy looks imported from mainline VGC and should be rewritten for Champions.

Fixes:

- Add `relative` to Pokemon cards.
- Default Items to Champions-only.
- Add expandable move/item rows.
- Add "common users", "top teammates", "sample role", and "countered by" context on detail pages.
- Update speed tier explanatory copy to match the Champions stat point system.

### Roster

Strengths:

- The roster model fits the app's differentiator: personal builds, status, VP, notes, stat points.
- Quick Add plus Full Form is a good split.
- StatPointEditor with speed benchmarks is a strong feature.

Issues:

- Without auth, this is a dead-end error state. It should have a sign-in prompt plus demo roster.
- Full Form has high friction for casual users: Pokemon, ability, nature, item, moves, stat points,
  status, VP, notes.
- Quick Add likely saves incomplete builds, but the UI does not make it clear what is missing or how
  to finish a build.
- Usage-aware ability/move/item sorting is good, but the app does not offer a "fill common set" or
  "start from common set" action.
- Stat points only use steppers in Roster; Calc uses sliders. This inconsistency adds friction.
- Speed benchmarks show closest benchmarks, not the practical "next thing I can outspeed if I add
  2/4/6 more points".

Fixes:

- Add unauthenticated value-preview: demo roster, sign-in CTA, and "import team to roster".
- Add "Use most common set" and "Clone from meta set" actions.
- Add build completeness indicators: missing item, less than 4 moves, no nature, no stat spread.
- Add direct numeric inputs or presets for stat points.
- Add "next useful speed target" suggestions in StatPointEditor.

### Teams

Strengths:

- Showdown import/export is present and strategically important.
- Team builder can edit roster entries inline, which is excellent.
- Mega selector accounts for multiple forms.
- Type coverage gives a basic health check.

Issues:

- The PRD calls for drag-and-drop, but the current form is click-to-select and does not support easy
  reordering.
- Empty-team state sends users to roster first, which is logical for the data model but too slow for
  both competitive and casual users who already have a Showdown paste.
- Showdown import requires a team name before preview. A competitive player expects paste first,
  parse immediately, then name later.
- Team analysis is mostly type coverage. Competitive team validation should also check roles:
  speed control, Fake Out, Protect count, Intimidate, redirection, priority, weather/terrain,
  Trick Room mode/checks, spread damage, setup, pivoting, and common meta weaknesses.
- No clear comparison against current top threats.

Fixes:

- Make "Import Showdown" the primary first-run path.
- Allow paste-first import preview, then team name/format confirmation.
- Add team order controls and drag/drop.
- Add role coverage and meta matchup audit alongside type coverage.
- Add "run damage benchmarks vs top 10 meta Pokemon" from a saved team.

### Draft Helper

Strengths:

- This is the most differentiated feature and matches a real competitive pain point.
- Saved Team vs Quick Pick is a good mode split.
- Opponent quick paste is a strong idea.
- Fast vs deep analysis, quota display, cache display, warnings, bring-four, leads, threats, calcs,
  and outcome logging all map to real player needs.

Issues:

- Home's `opp` parameter is not consumed, breaking the fastest path.
- Quick paste requires exact name matches. Competitive players often type aliases: "Rotom-W",
  "H-Arcanine", "A-Ninetales", "Farig", "Chomp", "Gambit".
- The app allows analysis with only one opponent Pokemon. That may be useful, but the UI should
  communicate confidence and make "6/6" the default target.
- "Fast analysis (Haiku)" and "Deep analysis (Sonnet)" expose implementation detail. Players need
  "Ladder quick read" and "Tournament prep".
- Results are long. During team preview, the player needs the answer pinned at top in a compact
  battle card: bring 4, leads, avoid, turn 1, must preserve, danger.
- Saving outcome only works for saved-team mode because `handleConfirmSave` requires
  `selectedTeamId`. Quick Pick users can analyze but cannot log the result cleanly.
- There is no opponent lead prediction, opponent likely bring-four, or side-by-side lead matchup
  matrix. Those are the most time-sensitive decisions.

Fixes:

- Implement URL hydration for `opp`.
- Add fuzzy name resolution using existing backend name resolver.
- Rename analysis modes to user outcomes.
- Require or strongly encourage 6/6 input with a confidence meter.
- Add compact "Team Preview Mode" result layout.
- Add opponent likely leads/bring-four.
- Allow Quick Pick analysis to be saved as an ad-hoc team or "session team".

### Damage Calculator

Strengths:

- Deterministic calc is a core tool competitive players expect.
- The three-column attacker/result/defender layout is good on desktop.
- Speed comparison inside results is useful.
- Query params for attacker/defender are already supported.

Issues:

- No saved team/set prefill. Competitive users should not re-enter spreads they already saved.
- No item, ability, burn, critical hit, Helping Hand, screens, terrain, spread target count,
  weather detail, or move-specific context visible in the UI.
- "All Moves" helps, but selecting a Pokemon should offer top usage moves and saved-set moves first.
- There is no "calc from Draft result" or "calc from Team builder benchmark" connection.

Fixes:

- Add "Load from roster/team" for attacker and defender.
- Add common battle modifiers as compact controls.
- Add deep links from Draft threats and Pokemon detail to prefilled calc.
- Add benchmark batches: run my six into top threats and top threats into my six.

### Cheatsheet

Strengths:

- Printable/pre-match reference is useful and differentiated.
- Stale detection after team edits is a strong trust feature.
- Share, copy link, regenerate, PDF export, and quick selection are good affordances.

Issues:

- Cheatsheets require roster/team data, so unauthenticated users cannot understand the value.
- Quick selection is useful, but there is no way to turn a session sheet into a saved team.
- The generated content likely needs a battle-first compact version, not only a printable full
  sheet.
- PDF export targets `id="cheatsheet-content"` and session sheets do not use that same id, so export
  may pick the wrong content when multiple sheets are present.

Fixes:

- Add a demo cheatsheet.
- Add "Save selection as team".
- Add "Battle card" and "Print sheet" display modes.
- Make PDF export target a passed element/ref, not a global id.

### Meta

Strengths:

- Usage list and trend data are highly relevant.
- Pokemon detail panel can connect meta research to roster/Pokedex/calc.

Issues:

- `GET /meta/latest` returned `[]`, so the Tier view can be empty even though usage/trends exist.
- `GET /usage?format=doubles` returned duplicate species from different sources, such as Incineroar
  appearing from both Pikalytics and Smogon. That can confuse rankings.
- The page copy says "Regulation M-A" and source names but does not show data freshness per source
  prominently enough for competitive trust.
- The meta page does not convert usage into actions: add to roster, find counters, show top teams,
  compare to my teams, or benchmark my team.

Fixes:

- Deduplicate usage by species/form and either merge sources or let users choose a source.
- Fix or seed tier snapshots for `/meta/latest`.
- Show source/date badges per row.
- Add actions: "Add to roster", "Counter this", "Calc vs this", "Teams using this".

### Matches

Strengths:

- Match logging is much richer than a basic W/L list: team, actual lineup, opponent six, leads,
  format, tags, close type, MVP, notes, stats.
- The "actual lineup differs from saved team" handling is good.

Issues:

- No replay URL field or replay import. Competitive players often review Showdown logs/replays.
- No opponent name, rating, round, tournament, game number, or Bo3 set grouping.
- Opponent leads are not captured, only my leads.
- Notes are unstructured; useful patterns need structured fields like loss reason, win condition,
  key turn, misplay, matchup label, and adjustment.
- Stats appear only after enough logs; no suggested insight or trend callout.

Fixes:

- Add replay URL/import.
- Add opponent name/rating, event/round, and Bo3 set grouping.
- Add opponent leads and selected four.
- Add structured tags for "lost to", "won by", "adjust next".
- Add insights: worst matchup, best lead pair, underperforming team, top loss reasons.

## Prioritized Improvements

### P0: Fix Broken and Trust-Damaging Flows

1. Fix Home quick draft: make `/draft?opp=` populate opponent slots and wire the icon button.
2. Add sign-in-aware empty states for Roster, Teams, Cheatsheet, Draft, and Matches.
3. Fix Meta tier empty state and usage deduplication.
4. Update stale hard-coded counts and regulation/week labels.
5. Fix Pokemon card absolute-position marker by adding a relative positioning context.

### P1: Make the Product Match Real Competitive Workflows

1. Make Showdown paste import the fastest team creation path.
2. Add fuzzy Pokemon name resolution to Draft and Match Log.
3. Add compact Draft "team-preview mode" with bring-four, leads, danger, and turn-one plan.
4. Add role coverage to Teams beyond type coverage.
5. Add saved team/set prefill to Calc.
6. Add replay URL and opponent leads to Match Log.

### P2: Improve Casual Adoption

1. Add demo roster/team/cheatsheet/draft without sign-in.
2. Add guided team templates: "balanced", "rain", "sun", "trick room", "hyper offense".
3. Add beginner-friendly explanations and role labels in Pokedex/Meta.
4. Add "use common set" and "why this set" actions.
5. Reduce nav complexity by grouping tools by task.

### P3: Differentiators Worth Building Next

1. Team matchup benchmark: my saved team vs top 10 meta threats.
2. Counter-team assistant from my roster.
3. Match log insights that suggest concrete team edits.
4. Team preview OCR or screenshot paste.
5. Public share pages for teams, not only cheatsheets.
